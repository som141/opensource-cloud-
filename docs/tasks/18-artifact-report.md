# 18. Artifact And Report

## Goal

Persist Worker preprocessing outputs to Object Storage and report artifact metadata back to the Backend Internal Worker
API.

This task is not OCR text extraction. The Worker stores processed images and preprocessing reports only.

## Read First

1. `README.md`
2. `docs/implementation-plan.md`
3. `docs/tasks/16-worker-preprocess-pipeline.md`
4. `docs/tasks/14-internal-worker-api.md`
5. `docs/worker/report-json-spec.md`

## Scope

1. Object Storage path rules.
2. Processed image upload.
3. Preview image upload.
4. Processing report JSON generation and upload.
5. Worker success callback with artifact object keys.
6. Failure mapping for artifact upload failures.

## Current Issue 79 Scope

Issue 79 implements the first real artifact/report integration:

1. `ObjectStoragePort.uploadBytes` is added.
2. `MinioObjectStorageClient` uploads generated bytes.
3. `ImageCodecAdapter` encodes output Mats as PNG.
4. `ProcessedImageSaveService` stores `processed.png`.
5. `PreviewImageSaveService` stores `preview.png`.
6. `ProcessingReportSaveService` stores `processing-report.json`.
7. `WorkerJobService` calls `reportSucceeded` after all required artifacts are uploaded.
8. Artifact upload failures report `ARTIFACT_UPLOAD_FAILED`.

## Output Paths

```text
processed/{projectId}/{jobId}/{itemId}/processed.png
processed/{projectId}/{jobId}/{itemId}/preview.png
processed/{projectId}/{jobId}/{itemId}/processing-report.json
```
