# 00. 저장소 기준 정리

## 목표

현재 저장소 상태를 확인하고, 이후 작업자가 따라야 할 문서 기준과 프로젝트 정체성을 고정한다.

## 먼저 읽을 문서

1. `README.md`
2. `.md`
3. `docs/implementation-plan.md`
4. `docs/tasks/README.md`

## 작업 범위

1. 기존 파일 확인
2. Git 상태 확인
3. 문서 디렉터리 기준 확인
4. 프로젝트 정체성 문구 정리
5. 작업 단위 문서 존재 여부 확인

## 작업 순서

1. `git status --short`로 변경 상태를 확인한다.
2. 루트 파일과 디렉터리 목록을 확인한다.
3. 기존 Markdown 파일을 모두 읽는다.
4. `docs/implementation-plan.md`의 전체 순서를 확인한다.
5. `docs/tasks/README.md`의 작업 순서를 확인한다.
6. 현재 작업이 어느 작업 단위에 해당하는지 정한다.
7. 작업 전 변경 범위를 짧게 정리한다.

## 산출물

1. 현재 저장소 상태 요약
2. 다음 작업 단위 선택
3. 필요 시 README 갱신 대상 메모

## 완료 기준

1. 기존 문서가 모두 확인되어 있다.
2. 작업 대상 문서가 정해져 있다.
3. 임의 구조 생성 없이 다음 단계로 넘어갈 수 있다.

## 금지 사항

1. 기존 사용자 변경 사항을 되돌리지 않는다.
2. 문서 확인 없이 코드를 생성하지 않는다.
3. 단순 이미지 리사이징 프로젝트로 표현하지 않는다.
