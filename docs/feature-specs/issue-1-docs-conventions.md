# 이슈 1. 문서와 작업 컨벤션

## 목적

프로젝트 초기 작업에서 사용할 GitHub workflow, 코드 컨벤션, API 응답 컨벤션을 문서로 고정합니다.

## 작업 범위

1. `docs/conventions/github-workflow.md` 작성
2. `docs/conventions/code-convention.md` 작성
3. `docs/conventions/api-response-convention.md` 작성
4. 이슈, 브랜치, 커밋, PR 규칙 정리

## 완료 기준

1. 기능 명세가 PR 본문에만 남지 않고 `docs/`에 저장됩니다.
2. 모든 새 작업은 이슈 기반 브랜치에서 진행합니다.
3. API 응답 형식은 공통 wrapper를 따릅니다.
